# Governance

Pitwall is currently maintained by `@buckeyes22`. The maintainer sets release
scope, reviews contributions, handles security reports, and makes final project
decisions. This is a solo-maintained project and does not promise an SLA.

Contributors are welcome to discuss changes in issues and pull requests.
Security boundaries, compatibility, data handling, supported deployment
surfaces, and governance changes should be documented in `docs/decisions/`.

Contributions use the Developer Certificate of Origin. Add a valid
`Signed-off-by` trailer to each commit; a contributor license agreement is not
required.

New maintainers may be added after a sustained record of technically sound and
respectful contributions. Having more than one maintainer is desirable, but it
is not a condition for publishing or using Pitwall.
