# Pitwall — SDLC Documentation

Comprehensive software-development-lifecycle documentation for **Pitwall**, a RunPod GPU broker.
Start with the [overview](00-overview.md) and [architecture](01-architecture.md); the per-subsystem
docs go deep on each component, grounded in the real source.

> Authoring note: the per-subsystem docs (02–13, 15, 16, 18, 19) were generated by an agentic
> documentation pass that read the actual source for each module; the cross-cutting docs (overview,
> architecture, security, testing strategy, this index) were authored directly. Treat the source
> tree as the source of truth — if a doc and the code disagree, the code wins (and the doc is a bug).

## Reading order

1. **[00 — Overview](00-overview.md)** — what Pitwall is, core capabilities, surfaces, technology.
2. **[01 — Architecture](01-architecture.md)** — layered view, the inference data flow, deployment
   topology, cross-cutting concerns, design principles.
3. Then dive into whichever subsystem you're working on.
