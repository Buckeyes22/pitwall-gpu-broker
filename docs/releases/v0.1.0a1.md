# Pitwall GPU Broker 0.1.0a1

This is the first public alpha of Pitwall, a self-hosted control plane for
routing and governing RunPod GPU workloads. It is a GitHub-first release:
Python distributions are attached to this release and the five supported
service images are published to GHCR. PyPI publication is intentionally
deferred.

## Included in this alpha

- REST, MCP, CLI, and TUI interfaces with scoped authorization.
- Capability-based routing across registered RunPod serverless and pod-lease
  providers.
- Pre-spend estimation, budgets, rate limits, audit records, lifecycle
  reconciliation, and kill-switch controls.
- Non-root images for the API, reconciler, webhook receiver, cost exporter,
  and MCP server.
- Reproducible wheel and source distributions, checksums, SPDX SBOMs, and
  GitHub build-provenance attestations.

## Important limitations

- This is pre-1.0 alpha software. Backward-compatible evolution is the goal,
  but breaking changes can occur under the documented
  [compatibility policy](../compatibility.md).
- Pitwall does not ship a GPU worker image. Operators supply and validate the
  image or existing RunPod endpoint that performs inference.
- RunPod is the supported provider for this alpha. Other provider integrations
  are not part of the public support claim; see the
  [support matrix](../support-matrix.md).
- Deployment is self-hosted. Operators remain responsible for infrastructure,
  provider credentials, workload data, model/image provenance, and spend.

## Installation and verification

Download the wheel or source archive and `SHA256SUMS` from this release. The
container images use the following namespace:

`ghcr.io/buckeyes22/pitwall-gpu-broker/<service>:v0.1.0a1`

Prefer immutable image digests in deployed configurations. Verify downloaded
artifacts against `SHA256SUMS` and the GitHub attestations before use. Start
with the repository README and deployment documentation for configuration.

Please report vulnerabilities privately according to the
[security policy](../../SECURITY.md). Use GitHub Discussions for support and
general questions.
